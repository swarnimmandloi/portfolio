Williams Companies:

Modernizing

Legacy Systems

Williams Companies, a major natural gas corporation, approached us to redesign their complex pipeline management system. This project presented unique challenges due to the intricate nature of the industry, the system's critical role, and the need to balance modernization with user familiarity.

My Role

Led the user-centric redesign of Williams Companies' legacy pipeline management system, focusing on creating a modern, efficient, and familiar user experience through agile development and stakeholder collaboration.

Problem Statement

The existing system, while functional, was built on outdated technology and design principles. Key issues included:

Aging user interface that no longer met modern standards.

Complex data management and visualization needs.

Resistance to change from long-time users accustomed to the existing system.

Need for improved efficiency in daily operations, particularly in accessing frequently used pages.

Challenges

Key challenges included:

User Resistance: Many users had been working with the existing system for 10-20 years, making significant UI changes challenging to implement and adopt.

Data Complexity: The system dealt with vast amounts of data from various sources, requiring careful consideration in design and implementation.

Scale: With thousands of pages in the system and individual users regularly accessing up to 50 pages daily, creating an efficient navigation and favorites system was crucial.

Critical Nature: As a system managing natural gas pipelines, any changes needed to ensure continued reliability and safety.

Approach

Our approach included the following strategies:

Design System Development: Created a comprehensive design system to ensure consistency across the vast number of pages. Focused on table designs, a critical component for data display, which could be easily implemented by developers.

Agile Methodology: Implemented weekly sprints to iteratively develop and refine features, allowing for continuous feedback and adjustments throughout the project.

User-Centric Design: Engaged closely with end-users to understand their daily workflows and pain points. Balanced modernization with familiarity to ease the transition for long-time users.

Stakeholder Engagement: Regularly communicated with Williams Companies leadership, presented design changes, and gathered feedback on proposed solutions, particularly for critical features like the dashboard and favorites system.

Key Solutions

The following solutions were implemented:

Redesigned Homepage/Dashboard: Transformed the homepage into an informative dashboard, incorporating key data visualizations and quick access to critical information.

Innovative Favorites System: Developed a robust favorites system to allow users to quickly access their frequently used pages, handling up to 50 favorites per user.

Enhanced Data Visualization: Improved the presentation of complex data through thoughtful table designs and other visualization techniques, ensuring that critical information was easily accessible and understandable.

Streamlined Navigation: Redesigned the navigation system to help users efficiently move through the thousands of pages in the system.

Outcomes

The project resulted in the following outcomes:


Successfully modernized the UI while maintaining familiarity for long-time users.

Implemented a favorites system that significantly improved daily workflow efficiency.

Received positive feedback from Williams Companies leadership on the redesigned dashboard and new features.

Created a scalable design system that allowed for consistent implementation across the entire platform.

Improved data visualization and accessibility, enhancing decision-making capabilities for pipeline management.

Learnings

Key learnings from the project included:

Balancing modernization with user familiarity is crucial in enterprise system redesigns.

Close collaboration with end-users and stakeholders is essential in complex, industry-specific projects.

Design systems play a vital role in ensuring consistency and efficiency in large-scale applications.

Innovative solutions, like the favorites system, can significantly impact user productivity in data-heavy environments.

Conclusion

This project showcases our ability to tackle complex enterprise challenges, balancing technical requirements with user needs and stakeholder expectations. By focusing on user-centric design and leveraging design systems, we successfully modernized a critical system while ensuring a smooth transition for its users.
